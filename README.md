# 文章服务平台

## 项目描述
文章服务平台为用户提供文章上传、点赞等服务，旨在满足高并发和高可用的需求。系统设计合理，能够高效处理大量用户请求，确保良好的用户体验。

## 技术栈
本项目采用以下技术栈构建：
- **Go**：作为后端编程语言，提供高性能的服务支持。
- **Gin**：使用 Gin 框架搭建稳定的 RESTful API 服务。
- **MySQL**：作为主数据存储，管理用户和文章数据。
- **Redis**：用于实现缓存机制，提高系统性能。
- **GORM**：作为 ORM 库，简化数据库交互。

## 系统架构设计
在系统设计阶段，我参与了整体架构的规划，重点负责用户模块和文章模块的后端开发。系统架构遵循分层设计原则，使各模块之间的耦合度降低，易于扩展和维护。

### 模块划分
- **用户模块**：处理用户注册、登录、信息管理等功能。
- **文章模块**：负责文章的上传、展示、点赞等操作。

## API 服务开发
基于 Gin 框架，我们搭建了稳定的 RESTful API 服务。API 的设计遵循标准 RESTful 风格，提供了清晰的接口文档，便于前端开发和第三方接入。

### 主要 API 接口
- `POST /articles`：上传新文章
- `GET /articles`：获取文章列表
- `POST /articles/:id/like`：给指定文章点赞
- `GET /users/:id`：获取用户信息

## 数据库优化
我们使用 GORM 进行数据模型的设计与实现，简化了与 MySQL 数据库的交互。通过合理的索引和查询优化，提升了数据库的性能，确保数据访问的高效性。

## 缓存集成
为提高系统的响应速度，我们集成了 Redis 来实现以下缓存机制：
- **用户登录态缓存**：减少频繁的数据库查询，提高用户登录的响应速度。
- **热门文章点赞缓存**：缓存热门文章的点赞数，减少对数据库的压力，提升系统的整体性能。

## 如何运行项目
1. 确保已经安装 Go、MySQL 和 Redis。
2. 克隆项目代码：
   ```bash
   git clone git@github.com:KinoHui/go-articleapp.git
   cd go-articleapp
   ```
3. 配置数据库连接和 Redis 设置。
4. 运行项目：
   ```bash
   go run main.go
   ```
